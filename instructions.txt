PROJECT OVERVIEW
----------------
SkillForge AI is a comprehensive AI-powered Educational Technology and Career Growth platform
featuring adaptive learning, role-based roadmaps, AI tutoring, career analytics, and social features.

TECHNOLOGY STACK
----------------
Backend:  Node.js + Express.js + MongoDB + Mongoose
Frontend: React 18 + Zustand + Tailwind CSS + Framer Motion
Auth:     JWT + Passport.js (Google/GitHub OAuth)
AI:       Gemini + Groq + Perplexity APIs
Email:    Nodemailer
Scheduler: node-cron
Real-time: Socket.io

                     5. KEY WORKFLOWS & USER JOURNEYS
═══════════════════════════════════════════════════════════════════════════════

WORKFLOW 1: USER REGISTRATION & ONBOARDING
───────────────────────────────────────────

START → Registration Page
  ↓
Choose Auth Method:
  ├─→ Email/Password → Enter credentials → Verify email
  └─→ OAuth (Google/GitHub) → Authorize → Auto-verify
  ↓
Create Account (POST /api/v1/auth/register)
  ↓
JWT Token Generated → Store in localStorage
  ↓
Redirect to Onboarding Page
  ↓
Onboarding Form:
  ├─→ Select Career Role (with AI suggestions)
  ├─→ Choose Skill Level (Beginner/Intermediate/Advanced)
  ├─→ Set Daily Study Time
  └─→ Select Preferred Language
  ↓
Save Preferences (POST /api/v1/users/onboarding)
  ↓
Generate Initial Study Plan
  ↓
Redirect to Dashboard
  ↓
END

WORKFLOW 2: LEARNING A TOPIC
─────────────────────────────

Dashboard → View Enrolled Roadmaps
  ↓
Select Roadmap → View Topics List
  ↓
Click Topic → Topic Page Loads
  ↓
Topic Page Components:
  ├─→ Video Player (YouTube videos in EN/TA/HI)
  ├─→ Markdown Documentation
  ├─→ Code Examples
  └─→ AI Teacher Button
  ↓
User Actions:
  ├─→ Watch Video → Track Time (TimeTrackingSession)
  ├─→ Read Documentation
  ├─→ Ask AI Teacher (POST /api/v1/ai/teacher)
  └─→ Generate Practice Quiz (POST /api/v1/ai/tutor/generate)
  ↓
Take AI-Generated Test
  ↓
Submit Answers (POST /api/v1/tests/submit)
  ↓
Check Score → Compare with Passing %
  ↓
IF Passed:
  ├─→ Mark Topic Complete (POST /api/v1/topics/:id/complete)
  ├─→ Award XP Points (badge system)
  ├─→ Update Roadmap Progress
  └─→ Update Heatmap Calendar
ELSE:
  └─→ Review Incorrect Answers → Retry Test
  ↓
END

WORKFLOW 3: CAREER PROFILE ANALYSIS
────────────────────────────────────

Career Page → Add LeetCode/GitHub Username
  ↓
Verify Username (POST /api/v1/users/verify-leetcode or verify-github)
  ↓
IF Valid → Trigger Analysis
  ↓
LEETCODE ANALYSIS (POST /api/v1/career/leetcode/analyze):
  ├─→ Fetch user stats from LeetCode GraphQL API
  ├─→ Calculate: Total Solved, Easy/Medium/Hard
  ├─→ Get Ranking, Acceptance Rate
  ├─→ Analyze Submission Calendar
  ├─→ AI Skill Assessment (via Gemini)
  └─→ Store in CareerAnalysis model
  ↓
GITHUB ANALYSIS (POST /api/v1/career/github/analyze):
  ├─→ Fetch repos via GitHub API
  ├─→ Get: Stars, Forks, Commits, Languages
  ├─→ Analyze contribution graph
  ├─→ Top repositories detection
  ├─→ AI Code Quality Assessment
  └─→ Store in CareerAnalysis model
  ↓
Display Insights:
  ├─→ Skill Level Score
  ├─→ Strengths & Weaknesses
  ├─→ Personalized Recommendations
  └─→ Activity Heatmaps
  ↓
Award XP & Badges:
  ├─→ "First Analysis" badge
  ├─→ "5+ GitHub Repos" badge
  └─→ "100+ LeetCode Problems" badge
  ↓
END

WORKFLOW 4: AI TUTORING SESSION
────────────────────────────────

User → Navigate to AI Chat Page
  ↓
Select AI Mode:
  ├─→ AI Teacher (Explanations)
  ├─→ AI Tutor (Practice)
  ├─→ AI Mentor (Career)
  └─→ AI Interviewer (Mock Interview)
  ↓
[TEACHER MODE]
  ├─→ Ask Question (POST /api/v1/ai/teacher)
  ├─→ Backend selects AI provider (Gemini → Groq → Perplexity)
  ├─→ Send prompt with context
  ├─→ Receive structured response
  ├─→ Store in AIChatSession
  └─→ Display formatted answer
  ↓
[INTERVIEWER MODE]
  ├─→ Start Interview (POST /api/v1/ai/interviewer/start)
  ├─→ AI generates technical questions
  ├─→ User submits answers (POST /api/v1/ai/interviewer/answer)
  ├─→ AI evaluates answer quality
  ├─→ Provides feedback and suggestions
  └─→ Generate final interview report
  ↓
END

WORKFLOW 5: DAILY STUDY PLAN GENERATION
────────────────────────────────────────

CRON Job (12:00 AM daily) → schedulerService.js
  ↓
For Each Active User:
  ↓
Gather User Data:
  ├─→ Enrolled Roadmaps
  ├─→ Completed Topics
  ├─→ Skill Level
  ├─→ Daily Study Time Preference
  └─→ Recent Activity
  ↓
AI-Powered Plan Generation:
  ├─→ Calculate topics to cover
  ├─→ Estimate time per topic
  ├─→ Balance difficulty
  ├─→ Consider dependencies
  └─→ Add variety (videos, docs, quizzes)
  ↓
Create DailyStudyPlan Document
  ↓
Send Email Reminder (emailService.js):
  ├─→ Subject: "Your Study Plan for Today"
  ├─→ List topics to study
  ├─→ Estimated time
  └─→ Motivational message
  ↓
User Accesses Dashboard → View Today's Plan
  ↓
Complete Plan Items → Mark as done
  ↓
Update Progress → Award XP
  ↓
END

  └─→ Add to "My Applications"
ELSE:

  ↓
Track Application Status:
  ├─→ New

  ↓

WORKFLOW 7: ADMIN CONTENT MANAGEMENT

Admin Login → Admin Dashboard
  ↓
Platform Statistics:

  ├─→ Topics Completed Today

  ↓

  ↓

──────────────────────────────

├─→ Topics completed

└─→ Community contributions

Displays as calendar heatmap with color intensity

MULTI-LANGUAGE SUPPORT:
───────────────────────

├─→ English (EN)

└─→ Hindi (HI)

User selects preferred language in settings

RESUME ATS SCORING:

   ├─→ Keyword matching
   ├─→ Format compliance
   ├─→ Content quality
   └─→ Suggestions

5. Detailed recommendations

                            9. SECURITY MEASURES
═══════════════════════════════════════════════════════════════════════════════

AUTHENTICATION:
───────────────
├─→ JWT tokens with 7-day expiration
├─→ Bcrypt password hashing (10 rounds)
├─→ OAuth 2.0 for Google/GitHub
├─→ Email verification required
└─→ Password reset via email token

AUTHORIZATION:
──────────────
├─→ Role-based access (User, Recruiter, Company, Admin)
├─→ Protected routes middleware
├─→ Resource ownership verification
└─→ Admin-only endpoints guarded

API PROTECTION:
───────────────
├─→ Rate limiting (100 req/15min general, 20 req/min for AI)
├─→ CORS configured for frontend origin
├─→ Helmet security headers
├─→ XSS protection
├─→ Input validation with express-validator
└─→ SQL injection prevention (Mongoose ORM)

DATA PROTECTION:
────────────────
├─→ Sensitive data encryption
├─→ Environment variables for secrets
├─→ HTTPS in production
└─→ Regular security audits
